Tree Data Structure

Detailed explanation, analysis, and implementation.

What is a Tree?

A Tree is a non-linear, hierarchical data structure consisting of nodes connected by edges. It simulates a hierarchical tree structure, with a root value and subtrees of children with a parent node.

Key Terminologies

  • Root: The topmost node of the tree.
  • Parent: A node that has an outgoing edge to another node.
  • Child: A node that has an incoming edge from another node.
  • Leaf: A node with no children.
  • Depth: Number of edges from root to the node.
  • Height: Number of edges on the longest path from the node to a leaf.

Types of Trees

1. Binary Tree

A tree where each node has at most 2 children (left and right).

2. Binary Search Tree (BST)

A binary tree with the property: Left child < Parent < Right child. This allows for fast searching.

3. AVL Tree

A self-balancing Binary Search Tree where the difference between heights of left and right subtrees cannot be more than one.

Tree Traversals

Traversal Order Description
InOrder Left -> Root -> Right Visits nodes in ascending order for BST.
PreOrder Root -> Left -> Right Used to create a copy of the tree.
PostOrder Left -> Right -> Root Used to delete the tree (visit children before parent).
LevelOrder Level by Level Breadth-First traversal.

Code Implementation

Structure of a Binary Tree Node.

C++
struct Node {
    int data;
    Node* left;
    Node* right;

    Node(int val) {
        data = val;
        left = right = nullptr;
    }
};

Node* root = new Node(1);
root->left = new Node(2);
root->right = new Node(3);
Java
class Node {
    int data;
    Node left, right;

    Node(int item) {
        data = item;
        left = right = null;
    }
}
Python
class Node:
    def __init__(self, key):
        self.left = None
        self.right = None
        self.val = key

root = Node(1)
root.left = Node(2)
root.right = Node(3)

Applications

  • File Systems (Directory structure).
  • DOM (Document Object Model) in HTML.
  • Database Indexing (B-Trees).
  • Compilers (Syntax Trees).
  • Auto-correction and spell checking (Tries).
